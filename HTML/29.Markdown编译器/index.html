你打算怎么实现该项目？
我将该项目分成了两大模块：
前端布局
后端翻译
现在只实现了最为简单的前端布局这一部分，有关层叠式代码块的实现这一小块内容仍然保留探究，因为不同语言之间的变量、函数、关键字应该如何分配颜色？缩进功能又应该如何实现？需不需要将代码切割成一个一个单词的形式？这些单词又需不需要用单独的 div 标签存储？诸如此类的种种问题，导致我仍然未能立马动工，如果不慎重的思考这一难题，到时候编码就会遇到很多坑爹的 Bug！
关于前端布局的主要思路
要求：
必须要有很方便的扩展性，即单独完整的实现 css 样式文件和必须的 js 文件，以这些作为 markdown 主题渲染的核心内容，到时候只要当前目录下存在这两个文件时，所有的 html 文件都能自动的使用这些效果
关于 css 样式文件，项目需要用到与主题相关的属性值，我都用一个单独的 css 文件，封装一个个的变量，将来如果我某段时间百感交集时，就可以通过改变这些变量的值，达成统一修改 markdown 主题的效果，非常方便
